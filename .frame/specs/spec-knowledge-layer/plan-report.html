1 · What & why

Frame's spec archive already records the project's most expensive knowledge — what was done, why that path won (including rejected alternatives), and what actually happened — but nothing delivers it to an agent at the moment it matters. Agents re-do archaeology every session and can confidently re-propose ideas that were already tried and rejected. Advisory text in AGENTS.md is not a mechanism: whether the agent honors it depends on memory, every session, every time.

The layer makes delivery deterministic without polluting context: spec artifacts are distilled into a per-spec digest.md and compiled into a derived, gitignored .frame/index/spec-index.json with two views (topic → specs, file → chronological history). Delivery uses two guaranteed channels — Claude Code hooks for unpredictable moments (an Edit lands on a storied file) and Frame-composed prompts for predictable ones (spec.new gets the full catalog, spec.plan reads footprint history, workers start preloaded). Injection is bound to the moment of intent (Edit/Write), never exploration (Grep/Read) — the shape that survived the memory-bloat discussion. The compact layer is an entry point only: on a hit the agent descends the raw chain (spec → plan → tasks → outcome) and relays the relevant history to the user in 1–2 sentences.

3 · Decision story

Business decisions

D1 · What is the default edit-time injection mode?
chosen
Full content (budgeted)
History lands in context unconditionally; cost bounded by dedup + overflow-to-pointer.
Signal-only
Cleanest context, but reading becomes the agent's choice — weakens the "cannot miss it" guarantee.
Why: the entire spec exists because advisory/optional mechanisms proved unreliable; signal-only survives behind FRAME_SPEC_HINT_MODE=signal for dogfooding comparison.
D2 · Does the Frame UI file-history panel ship in this spec?
chosen
Follow-on spec
v1 footprint stays main-process + scripts + templates; UI builds later on the on-disk index contract.
Include renderer work now
One-shot delivery, but grows footprint into renderer/CSS and delays the core layer.
Why: the core layer is agent-facing and independently valuable; the index file is a stable contract a UI spec can consume unchanged.
D3 · Are inventory hygiene and legacy backfill part of this spec?
chosen
Both in this spec
Spec grows, but the layer ships working on a clean, populated archive.
Mechanism only
Smaller spec, but the first index derives from probe debris and stale phases — noise on day one.
Why: the index is only as good as the archive it derives from, and this layer is what finally makes spec hygiene load-bearing.

Technical decisions

D4 · Where does digest authoring live, given no spec.done command exists? from evidence drift
chosen
spec.implement / WORKER.md final-task step
No new command or UI; digest written in the same turn the last task completes, outcome freshest.
New /spec.done command
Explicit closing ritual, but adds command infra + phase mechanics for one step.
Why: evidence pass showed done is auto-derived from task completion (derivePhase, specManager.js:216) — there is no closing command to hook into, and inventing one just for the digest is scope the flow doesn't need.
D5 · Is spec-index.json tracked in git?
chosen
Gitignored + lazy rebuild (ensureFresh)
Fresh clones need one rebuild before hints appear; everything else regenerates on demand.
Tracked
Hook works instantly on fresh clones, but generated JSON enters every closing commit and conflicts on parallel branches.
Why: the engineering findings documented exactly this trap with STRUCTURE.json (most-conflicting file in the repo); a derived artifact must stay derived.
D6 · Where is the hook registered?
chosen
Project .claude/settings.json (merge-safe write)
Tracked and team-wide — every teammate and worktree gets the hook with zero setup.
settings.local.json
Keeps the user's repo untouched, but every machine/worktree needs separate install — weakens "always works".
Why: determinism was the founding requirement; user projects get a read-modify-write that preserves existing keys, gated on ai_tool: claude-code, with manual instructions surfaced on unparseable JSON.
