## How to operationalize this

### Step 1 — Twitter Lists (private)

Create 2 private lists:
- `frame-tier-1-2` — Anthropic + power users. Highest engagement priority. Check daily.
- `frame-tier-3-4-5` — Broader. Skim, don't drown.

### Step 2 — TweetDeck / X Pro columns

Recommended column layout:
1. Tier 1-2 list (real-time)
2. `"claude code"` keyword
3. `"CLAUDE.md"` keyword
4. `"agentic coding"` keyword
5. Frame mentions (`@kaanozhan` mentions + `Frame` keyword filtered)
6. `"spec kit"` OR `"spec-driven"`

### Step 3 — Discipline

- Tier 1-2: see every post.
- Tier 3-5: skim via keyword filter, don't try to read everything.
- Twice a day TweetDeck check: morning + evening, ~10 minutes each.
- Don't follow these accounts publicly if you don't want to (private lists are invisible).

### Step 4 — Engagement order

When multiple tweets are queued for reply:
1. Tier 1-2 first (highest leverage, fastest decay)
2. Tweets <30 min old (algorithm timing)
3. Tweets with <50 replies (less noise)
4. Tweets where you can add a specific, non-obvious thing

## Don't-follow / Don't-engage list

- Hyper-political accounts even if AI-adjacent (off-topic dilution)
- "AI guru" accounts that don't ship anything (audience overlap zero)
- VC pundits (wrong audience for an indie OSS tool)
- Anyone in active flame wars (don't get tagged in)
